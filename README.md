# Excel API

**FastAPI** service that generates Excel (`.xlsx`) files from JSON: **Daily report** (`/reports/daily-report`) and **Mileage report** (`/reports/mileage-report`). The **daily report** replaces the legacy `/generate-excel` flow: column **A** (month days) is highlighted for weekends and holidays, percentages use the template format, the footer includes the last business day of the month, and the suggested filename is `report_<month>_<timestamp_utc>.xlsx`. The **mileage report** shares `month` (1–12), `holidays`, and the `employee` block (including **`vehicle_plate`**); `company` includes **`address`**, and `entries` use trip fields and **`distance_km`** (see Mileage report section).

## Requirements

- Python **3.11** or newer
- [uv](https://docs.astral.sh/uv/) (dependency and environment manager)
- `.xlsx` templates and JSON mappings under `src/reports/daily_report/` and `src/reports/mileage_report/` (see [Configuration](#configuration))

## Installation

From the repository root:

```bash
uv sync
```

With development dependencies (tests, coverage, etc.):

```bash
uv sync --group dev
```

Equivalent: `make install` (runs `uv sync`).

### `POST /reports/daily-report`

Generates the **daily report** (template: `src/reports/daily_report/template.xlsx`, mapping: `mapping.json`). Behaviour matches the legacy `/generate-excel` flow: header, rows, footer (incl. last business day), percentages as **%** with highlight when &lt; 100%, column A highlighting (holiday &gt; weekend), **invalid** `holidays` values in **1–31** are **ignored**, extra fields are **ignored**.

- **`month`:** integer **1–12** (required).
- **`employee`:** optional for validation but required by the shared schema for footer data.

**Minimal example:**

```json
{
  "company": { "name": "ACME", "tax_id": "123456789", "address": "1 Main St" },
  "employee": { "name": "Jane Doe", "tax_id": "987654321", "address": "2 Main St" },
  "month": 3,
  "entries": [],
  "holidays": [25]
}
```

**Full example:**

```json
{
  "company": {
    "name": "My Company",
    "tax_id": "123456789",
    "address": "1 Main St"
  },
  "employee": {
    "name": "John Doe",
    "address": "50 Central Ave",
    "tax_id": "222333444"
  },
  "month": 3,
  "entries": [
    {
      "day": 1,
      "description": "Meeting",
      "location": "Lisbon",
      "start_time": "09:00",
      "end_time": "10:00",
      "percentage": 100
    }
  ],
  "holidays": [5, 25]
}
```

**`curl`:**

```bash
curl -X POST http://localhost:8000/reports/daily-report \
  -H "Content-Type: application/json" \
  -d '{
    "company": { "name": "Test", "tax_id": "123", "address": "Addr" },
    "employee": { "name": "Jane", "tax_id": "999", "address": "Addr" },
    "month": 3,
    "entries": [{ "day": 1, "description": "Activity", "percentage": 100 }],
    "holidays": [25]
  }' \
  -o daily_report.xlsx
```

**Success:** status **200**, **XLSX** body, suggested filename: `report_<month>_<timestamp_utc>.xlsx` (e.g. `report_3_20260321T120000Z.xlsx`).

### `POST /reports/mileage-report`

Generates the **mileage report** (template under `src/reports/mileage_report/`).

**`company`** (required fields per schema: `name`, `tax_id`, `address`):

- `name`, `tax_id` — header.
- **`address`** — company address in cell **B5**.

**`month`** (1–12):

- Written to Excel as the **English month name** (e.g. `3` → `March`).

**`entries`** (trip rows):

| JSON field | Excel column |
|------------|----------------|
| `day` | A (day) |
| `origin` | B |
| `destination` | C |
| `description` | D (justification) |
| `distance_km` | F (distance) |

`distance_km` may be a number or string. **Comma is the decimal separator** (European style): `"12,5"` → 12.5; `"363,000"` → **363** with three decimal places (not 363 thousand). Thousands in the integer part may use **dot**: `"1.234,56"`.

**`employee`** (optional): `name`, `address`, `tax_id`, **`vehicle_plate`** (footer “Received by” / vehicle).

**`holidays`:** integers **1–31**; invalid values are ignored.

**Minimal example:**

```json
{
  "company": { "name": "ACME", "tax_id": "123456789", "address": "1 Main St" },
  "employee": { "name": "Jane", "tax_id": "999", "address": "Addr" },
  "month": 3,
  "entries": [],
  "holidays": [25]
}
```

**Full example:**

```json
{
  "company": {
    "name": "My Company",
    "address": "50 Central Ave",
    "tax_id": "123456789"
  },
  "employee": {
    "name": "John Doe",
    "address": "50 Central Ave",
    "tax_id": "222333444",
    "vehicle_plate": "34-XY-56"
  },
  "month": 3,
  "entries": [
    {
      "day": 1,
      "origin": "Braga",
      "destination": "Lisbon",
      "description": "Travel for tasks",
      "distance_km": "363,000"
    }
  ],
  "holidays": [5, 25]
}
```

**`curl`:**

```bash
curl -X POST http://localhost:8000/reports/mileage-report \
  -H "Content-Type: application/json" \
  -d '{
    "company": { "name": "Test", "tax_id": "123", "address": "Street X" },
    "employee": { "name": "John", "tax_id": "999", "address": "Addr", "vehicle_plate": "34-XY-56" },
    "month": 3,
    "entries": [{
      "day": 1,
      "origin": "Braga",
      "destination": "Lisbon",
      "description": "Trip",
      "distance_km": "100"
    }],
    "holidays": [25]
  }' \
  -o mileage_report.xlsx
```

**Success:** status **200**, **XLSX** body, suggested filename: `report_<month_number>_<timestamp_utc>.xlsx` (month **1–12** in the filename for ASCII headers).

## Common errors

- **422:** JSON body does not match the schema (e.g. missing `month`, invalid `month`) — JSON validation response  
- **500:** Failed to generate the file (template, processing) — JSON with `error`, `message`, `status`  

## Configuration

Each report uses a **template** `.xlsx` and a **`mapping.json`** under `src/reports/<module>/`. Colour constants (weekends, holidays, percentage &lt; 100%) are in `src/core/config.py`.

| Variable | Description |
|----------|-------------|
| `LOG_LEVEL` | Log level (e.g. `INFO`) |
